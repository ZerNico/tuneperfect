## Direction findings (maintainer options, not defects)

- **Redis-backed signaling**: `src/signaling/publisher.ts:9-12` documents that WebRTC signaling is in-memory and breaks with >1 API replica; Redis is already a dependency. The single hard blocker to horizontal scaling.
- **Account deletion & data export**: no `deleteMe` endpoint exists (GDPR right-to-erasure gap); schema already cascades on every user FK.
- **Club join requests / notifications**: invites are strictly one-directional and polled; weaker demand evidence — hold.

## Findings considered and rejected

(So nobody re-audits these.)

- **Drizzle RC pin → "upgrade to stable"**: no stable 1.0 exists (npm latest 0.45.2; 1.0.0 at beta.22) and the code uses the 1.0-line relational query syntax. Revisit at GA.
- **Club invite check-then-act race**: guarded by `onConflictDoNothing` with an explanatory comment (`src/club/service.ts:100-110`); by design.
- **CSRF tokens on REST routes**: origin/referer plugin rejects requests missing both headers + `SameSite=Lax` cookies; token plumbing adds complexity for ~no gain.
- **Refresh-token timing attack**: lookup is by SHA-256 hash of an attacker-supplied 256-bit token; not practically exploitable.
- **Avatar cache-buster `t` param "not cryptographic"**: it's a cache-buster, not a security control.
- **`getAvailableClubs` "missing auth"**: requires the lobby token, which is the intended capability; by design.
- **Pagination on club/invite lists; member over-fetch as perf issue**: lists are naturally small at current scale; the security-relevant slice (emails) is handled by plans 001/002.
- **Defensive null-checks on `.returning()[0]` / `isOwnerOrAdmin(undefined)`**: functionally correct today; pure style.
- **`bun audit` critical (protobufjs)**: enters via `posthog-js` in `apps/app`/`game`/`web` — outside the `apps/api` audit scope. Flag for a frontend-scoped pass.
